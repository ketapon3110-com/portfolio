ALOBABY ブランドサイト リニューアル

主軸：構造再設計 / パフォーマンス改善（約1年）

背景

ブランド横断の共通設計により、ブランド体験の最適化が難しく、 TOP〜商品詳細の直帰率が高い状態。商品詳細→カート→注文完了の遷移も伸び悩み。

体制

リード：自分（設計・実装主導） / サポート：ディレクター1名。 関係部署を含め全19名規模で推進。

自分の担当

方針立案（事前に方向性を設計）→社内提案・調整→UI設計（全ページ）→ HTML/CSS/JavaScript/Liquid実装→パフォーマンス改善。 最終意思決定は上司・社長承認。

使用技術

HTML / CSS / JavaScript / Liquid（ECForce）

構造再設計

ブランド間クロスの優先度を下げ、TOP〜商品詳細をブランドごとに分離。 体験とルール（見せ方・導線）をブランド単位で最適化できる構造へ変更。

主な施策

画像のWebP化、読み込み順序の最適化、不要なJavaScriptの整理、 レンダリングを止める要因の削減、レイアウトずれ対策（画像サイズ明示）など。

コード例（短い抜粋）
// 例：画像のレイアウトずれ対策（CLS抑制）
// 改善前：サイズ未指定で読み込み後にズレる
// <img src="hero.jpg">

// 改善後：width/heightを明示し、ズレを抑える
// （実際はテンプレート都合に合わせてLiquid側で出し分け）
<img src="/assets/img/hero.webp" width="1200" height="800" loading="eager" decoding="async" alt="">

※固有情報は伏字・置換でOK。差分と狙いが伝われば十分。
