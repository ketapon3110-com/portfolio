GA4計測設計・分析基盤構築

Analysis / Direction / Tracking

URL：社内利用のため非公開
自社EC全体の「どの施策が、どこまで届き、どこで離脱しているか」を見えるようにするため、GA4を用いた計測設計とダッシュボード基盤を構築した。
使用ツール
Google Analytics 4 / Looker Studio / ECForce / Spreadsheet / Excel
言語
JavaScript / SQL
期間
初期設計〜ダッシュボード公開まで：数か月
以降は運用しながら継続的に改善
対応範囲
計測方針の策定 / イベント設計 / パラメータ設計 / 実装ディレクション / テスト / ダッシュボード要件定義 / レポート運用支援
概要
自社ECサイトで行うUI改善やキャンペーン、CRM施策の結果が「どのページで、どの流れの中で効いているか」を共通の指標で見られるようにするため、GA4での計測設計とレポート基盤を整えた。
これにより、「作って終わり」の状態から、数値を見ながら改善サイクルを回せる状態へと移行した。
背景と課題
以前は、ページ単位のアクセス数やコンバージョン数は見られたものの、
・どの流入経路から来たユーザーが
・どの導線を通り
・どこで離脱しているのか
を一つの流れとして見ることが難しかった。

そのため、
・LPを作り替えても、どこが効いたのか判断しづらい
・LINEやメルマガからの流入が、どのくらい購入につながっているか見えにくい
・施策ごとのレポート形式が担当者ごとにバラバラで、比較しづらい
といった状態になっていた。

Web制作とCRM施策を横断して改善するには、「同じ物差し」で見られる分析基盤が必要だと考え、GA4を軸にした計測設計とレポート整備に着手した。
ゴール
・施策ごとの成果が、決まった指標で比較できる状態にすること
・流入〜回遊〜購入までの一連の動きを、ファネルとして把握できるようにすること
・Web制作・CRM・広告など、関係者全員が同じレポートを見て議論できる状態にすること

をゴールに置き、「誰が見ても同じ意味で理解できる数値」を増やしていくことを意識して設計した。
アプローチ

1. 計測の棚卸しと項目整理

まず、既存で取れている数値と、意思決定に本当に必要な数値を整理した。
・売上会議や施策報告でよく使う指標
・UI改善やCRM施策の検証に必要な指標
を洗い出し、紙やスプレッドシート上で「どの画面で、何を、どう測るか」を一覧化した。

2. イベント・パラメータ設計

次に、GA4で扱うイベントと、その中に含める情報を設計した。
・商品詳細閲覧、カート投入、購入開始、購入完了といった基本の流れ
・バナークリック、CVボタンクリック、スクロール率などの行動
・LINE・メルマガ・キャンペーンごとの識別情報
をひとつひとつ定義し、命名ルールと値の入れ方を整理した。

3. 実装とテスト

ECForce上でのタグ埋め込みや、既存テンプレートとの兼ね合いを踏まえながら、JavaScriptでイベント送信の実装を行った。
テスト用ビューを用意し、
・想定通りのイベントが発火しているか
・不要な二重計測が起きていないか
・パラメータに誤った値が入っていないか
を一つずつ確認し、ログを見ながら調整した。

4. ダッシュボード設計と運用

最後に、Looker Studioを使って、施策検証用のダッシュボードを整えた。
・流入元別のファネル（流入→閲覧→カート→購入）
・LPごとの離脱ポイント
・LINE・メルマガ施策ごとの結果比較
など、現場で実際に使う単位でページを分け、「毎週の定例でそのまま使える」ことを意識して構成した。
実装で意識したこと
計測を増やし過ぎて管理できなくなることを避けるため、
・イベント名を増やしすぎない
・パラメータの意味を明確にして重複を避ける
・キャンペーン名や施策IDの付け方をルール化する
ことを特に意識した。

また、将来の画面改修や施策追加にも耐えられるよう、
・テンプレート単位で共通のイベントを使い回す
・サイト側のHTML構造が少し変わっても壊れにくいセレクタを使う
といった点にも注意し、保守しやすい実装を心がけた。

構造化データやページ構造の見直しなど、技術的なSEO改善と合わせて進めることで、
表示速度・検索評価・行動データの三つを同時に整えることも意識した。
My Role
計測の方針整理からイベント・パラメータ設計、実装ディレクション、テスト、ダッシュボード要件定義まで、一連の流れを担当した。
Web制作・CRM・広告・経営層など関係者の「見たい数値」をヒアリングし、それぞれの要望を共通の指標に落とし込む部分をリード。

導入後も、実際の運用で出てきた「この数値も欲しい」「ここは見ていないので簡略化したい」といった声を受けて、計測内容とレポート構成の微調整を継続的に行った。
数値・成果

施策検証のスピード向上

・LP改修やキャンペーン実施のたびに個別で資料を作成する必要がなくなり、
　定例会ではダッシュボードを見ながら議論できるようになった。

部門をまたいだ共通言語の形成

・Web制作、CRM、広告、経営層が同じ画面を見ながら話せるようになり、
　「なんとなく良さそう」から「この指標がこう変わったので次はここを触る」という会話に変わった。

改善サイクルの定着

・施策ごとの成果が見えるようになったことで、
　小さな改善を短い周期で繰り返すスタイルが定着した。
・継続率改善やF2転換率改善など、他施策の検証にもこの基盤を流用できるようになった。
